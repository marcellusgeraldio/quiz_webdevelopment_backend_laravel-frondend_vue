Proses dimulai ketika pengguna berinteraksi dengan antarmuka Vue, seperti ketika mereka mencari nama produk di kolom pencarian atau menekan tombol navigasi halaman. Vue menanggapi interaksi ini dengan mengirimkan permintaan HTTP ke endpoint API Laravel, yang biasanya menggunakan library seperti Axios. Data parameter seperti kata kunci pencarian, kategori yang dipilih, kolom pengurutan, dan nomor halaman yang sedang aktif dikirimkan bersama permintaan tersebut.


Controller mengirimkan permintaan ke Laravel di backend. Fitur Eloquent Query Builder berfungsi untuk menjalankan logika penyaringan data. Fitur ini menentukan apakah ada parameter pencarian yang dapat ditambahkan secara dinamis ke klausa filter (where). Setelah data difilter, Laravel menggunakan perintah order by untuk melakukan pengurutan sesuai keinginan pengguna. Selanjutnya, fungsi paginate digunakan untuk memotong data untuk membatasi jumlah baris yang dapat dikirim per halaman. Setelah proses olahan selesai, hasilnya dikembalikan ke Vue dalam bentuk JSON. Ini mengandung array data produk dan informasi meta pagination seperti jumlah halaman dan total data.


Data disimpan ke dalam state lokal komponen atau state management global setelah Vue menerima tanggapan JSON dari Laravel. Dengan sifat reaktif Vue, perubahan pada variable state yang berisi daftar produk akan secara otomatis memperbarui tampilan tabel di layar pengguna tanpa perlu mengulangi halaman. Informasi tentang halaman saat ini—atau halaman saat ini—juga diperbarui di state, yang kemudian dapat digunakan untuk menonaktifkan tombol Previous jika berada di halaman pertama atau tombol Next jika sudah mencapai halaman terakhir. Vue akan mengidentifikasi perubahan state saat pengguna mengubah filter atau berpindah halaman dan memulai pengambilan data ulang ke server.
